## 🎯 ΒΑΣΙΚΕΣ ΛΕΙΤΟΥΡΓΙΕΣ

### 📍 ΠΡΟΣΘΗΚΗ ΣΥΣΚΕΥΩΝ  
- ΚΛΙΚ: Επιλέξτε συσκευή από το αριστερό πάνελ (Router, Switch, PC, Server, DNS, Cloud/WAN, Firewall, Printer).  
- ΜΕΤΑΚΙΝΗΣΗ: Σύρετε (drag & drop) τις συσκευές μέσα στο workspace.  

### 🔗 ΣΥΝΔΕΣΗ ΣΥΣΚΕΥΩΝ  
1. Πατήστε **Connect** 🔗 στη γραμμή εργαλείων.  
2. Επιλέξτε 1η συσκευή.  
3. Επιλέξτε 2η συσκευή.  
4. Δημιουργείται αυτόματα σύνδεση ανάμεσά τους.  

### 🚦 AUTO ROUTING  
- Πατήστε **Routing** για αυτόματη ρύθμιση gateways προς τον router όπου είναι δυνατό.

## 🌐 ΠΡΟΚΑΘΟΡΙΣΜΕΝΑ ΜΟΝΤΕΛΑ JSON

### 📂 ΔΙΑΘΕΣΙΜΑ ΜΟΝΤΕΛΑ  

Πατήστε **Load Model** (📁) για να ανοίξει το παράθυρο επιλογής μοντέλων.  

| Μοντέλο | Αρχείο | Περιγραφή |
|---------|--------|-----------|
| **LAN** | `SimpleLan.json` | Μικρό LAN: Router + Switch + PCs/Servers σε 192.168.1.0/24 |
| **2 Υποδίκτυα** | `2subnetsswitch.json` | 2 υποδίκτυα πίσω από switch, ιδανικό για subnetting & routing |
| **LAN–WAN** | `Officewith2SubnetsaRouterInternet.json` | Γραφείο με router προς Internet (WAN) και 2 LAN subnets |

**Φόρτωση μοντέλου:**  
1. Πατήστε **Load Model**.  
2. Επιλέξτε το μοντέλο (π.χ. LAN–WAN).  
3. Φορτώνονται αυτόματα συσκευές, συνδέσεις και DNS records, σαν έτοιμο workspace.

## ⚙️ ΡΥΘΜΙΣΕΙΣ ΣΥΣΚΕΥΩΝ (ΓΕΝΙΚΑ)

Κάθε συσκευή έχει ιδιότητες: IP, Subnet Mask, Gateway, DNS Server, Domain Name (όπου υποστηρίζεται) και Status.  

- Όλες οι αλλαγές περνούν από βασικό έλεγχο εγκυρότητας (0–255 για IP octets, έγκυρες μάσκες κ.λπ.).  
- Σε μη έγκυρη τιμή εμφανίζεται μήνυμα σφάλματος και η ρύθμιση δεν εφαρμόζεται.  

### Πεδία / Έλεγχοι  
- **IP Address**: Μορφή x.x.x.x με κάθε octet 0–255 (π.χ. 192.168.1.10).  
- **Subnet Mask**: Επιτρέπονται μάσκες όπως 255.255.255.0, 255.255.0.0, 255.0.0.0 κ.ά.  
- **Gateway**: Αν δεν είναι 0.0.0.0, τυπικά είναι η IP του router στο ίδιο δίκτυο.  
- **DNS Server**: Έγκυρη IP DNS server (εσωτερικού ή εξωτερικού, π.χ. 8.8.8.8).  
- **Domain Name**: Ονομασία τύπου `host.local` που καταχωρείται και στον DNS Manager.  

***

## 🧭 ROUTER – ΠΟΛΛΑΠΛΕΣ ΔΙΕΠΑΦΕΣ

Ο router διαθέτει τουλάχιστον τρεις λογικές διεπαφές: WAN, LAN και LAN2.  

**Προεπιλεγμένες ρυθμίσεις router όταν προστίθεται:**  

- **WAN**  
  - IP: `N/A`  
  - Subnet: `255.255.255.0`  
  - Gateway: `0.0.0.0`  
  - DNS: `8.8.8.8`  

- **LAN (192.168.1.0/24)**  
  - IP: `192.168.1.1`  
  - Subnet: `255.255.255.0`  
  - Gateway: `0.0.0.0`  
  - DNS: `192.168.1.1`  

- **LAN2 (192.168.2.0/24)**  
  - IP: `192.168.2.1`  
  - Subnet: `255.255.255.0`  
  - Gateway: `0.0.0.0`  
  - DNS: `192.168.2.1`  
  - Κατάσταση: `enabled = true` (μπορεί να απενεργοποιηθεί από checkbox).  

Στο πάνελ router μπορείτε να ρυθμίσετε ξεχωριστά IP/Subnet/Gateway/DNS για WAN, LAN και LAN2, καθώς και να ενεργοποιήσετε/απενεργοποιήσετε τη LAN2.  

***

## 🔀 SWITCH

Ο switch υποστηρίζει unmanaged και managed λειτουργία.  

- **Unmanaged switch** (default):  
  - IP: `N/A`  
  - Subnet: `255.255.255.0`  
  - Gateway: `0.0.0.0`  
  - DNS: κενό  
  - Χρήση: απλό Layer 2, χωρίς IP διαχείρισης.  

- **Managed switch**:  
  - Αν δοθεί IP στο πάνελ, ρυθμίζονται και Subnet/Gateway/DNS όπως σε κανονική συσκευή.  

***

## 💻 STANDARD ΣΥΣΚΕΥΕΣ (PC, SERVER, PRINTER)

Standard συσκευές μοιράζονται κοινή λογική IP configuration.  

- **Χωρίς router στο workspace**:  
  - IP: τυχαία private 192.168.x.y  
  - Subnet: `255.255.255.0`  
  - Gateway: `0.0.0.0`  
  - DNS: `8.8.8.8`  

- **Με router (π.χ. 192.168.1.1)**:  
  - Η IP τοποθετείται μέσα στο LAN του router (192.168.1.x) αποφεύγοντας διπλοκαταχωρήσεις.  
  - Gateway: `192.168.1.1`  
  - DNS: συνήθως επίσης `192.168.1.1`, εκτός αν ορίσετε άλλο.  

Στο πάνελ μπορείτε να αλλάξετε IP, Subnet, Gateway, DNS και Domain Name.  
Όταν ορίσετε Domain Name και υπάρχει έγκυρη IP, δημιουργείται αυτόματα DNS A record.  

***

## 🌐 CLOUD / WAN

Η συσκευή Cloud/WAN αναπαριστά το εξωτερικό δίκτυο / Internet.  

- IP: `8.8.8.8`  
- Subnet: `255.255.255.0`  
- Gateway: `0.0.0.0`  
- DNS: `8.8.8.8`  

Μπορεί να προσαρμοστεί για σενάρια με άλλο πάροχο/UPSTREAM.  

***

## 🧱 DNS SERVER

Ο DNS server είναι ειδική συσκευή τύπου `dns` με δικό της σύνολο εγγραφών.  

**Προεπιλογές DNS server:**  

- IP: `192.168.1.53`  
- Subnet: `255.255.255.0`  
- Gateway: `192.168.1.1`  
- DNS: `192.168.1.53` (self)  

Στο πάνελ DNS μπορείτε:  
- Να προσθέσετε νέα records (host → IP).  
- Να διαγράψετε υπάρχοντα records.  
- Να δείτε ταξινομημένη λίστα records (Local, External, Custom).  

Ο DNS Manager διατηρεί έναν global πίνακα με:  
- Εξωτερικά ονόματα (π.χ. `google.com`, `github.com`).  
- Τοπικά ονόματα (`router.local`, `dns.local`, `printer.local` κ.λπ.).  
- Domain names που δώσατε σε συσκευές.  

***

## 🌐 DNS RESOLUTION – ΛΟΓΙΚΗ

Όταν μια συσκευή κάνει DNS query (αυτόματα ή μέσω Manual DNS):  

1. Ελέγχεται η τοπική DNS cache της συσκευής.  
2. Αν υπάρχει ρυθμισμένος DNS server (στο πεδίο DNS), γίνεται προσπάθεια χρήσης αυτής της DNS συσκευής.  
3. Αν δεν βρεθεί DNS συσκευή, γίνεται αναζήτηση στα global DNS records.  
4. Αν υπάρχει συσκευή με `domainName` ίδιο με το ζητούμενο, επιστρέφεται η IP της.  
5. Σε επιτυχία, η IP αποθηκεύεται στην cache και καταγράφεται RESPONSE στην κονσόλα, με οπτική κίνηση πακέτου DNS.  
6. Σε αποτυχία, εμφανίζεται QUERY failed με αντίστοιχο μήνυμα.

## 🔧 ΕΡΓΑΛΕΙΑ ΔΙΚΤΥΟΥ

### 🚀 PING  
1. Επιλέξτε συσκευή στο workspace.  
2. Πατήστε **Ping**.  
3. Επιλέξτε προορισμό (συσκευή ή IP).  
4. Η κονσόλα εμφανίζει τα αποτελέσματα ping και, αν χρειαστεί, ανοίγει αυτόματα.  

### 🛣️ ROUTE (Διαδρομή)  
1. Πατήστε **Route**.  
2. Επιλέξτε συσκευή πηγή.  
3. Επιλέξτε συσκευή προορισμό.  
4. Εμφανίζεται η διαδρομή με όλα τα hops (router, switches) και οπτικοποίηση στο workspace.  

### ✋ MANUAL DNS  
1. Πατήστε **Manual DNS**.  
2. Επιλέξτε συσκευή-πηγή.  
3. Επιλέξτε DNS server.  
4. Εισάγετε domain.  
5. Το αποτέλεσμα (επιτυχία/αποτυχία, IP προορισμού, πηγή εγγραφής) εμφανίζεται στην κονσόλα με animation DNS query/response.  

### ▶️ SIMULATION  
- Πατήστε **Simulate** για να ξεκινήσει ροή τυχαίων πακέτων στο δίκτυο.  
- Τα πακέτα εμφανίζονται οπτικά στις συνδέσεις, ενώ τα counters ενημερώνονται.  
- Πατήστε ξανά **Simulate** για παύση.

## 💾 ΑΠΟΘΗΚΕΥΣΗ & ΦΟΡΤΩΣΗ WORKSPACE

### Αποθήκευση (Save)  
1. Πατήστε **Save** (💾).  
2. Δώστε όνομα workspace.  
3. Δημιουργείται και κατεβαίνει JSON αρχείο με metadata, devices, connections και DNS records.  

### Φόρτωση (Load)  
1. Πατήστε **Load** (📂).  
2. Επιλέξτε JSON workspace.  
3. Το τρέχον workspace καθαρίζεται και φορτώνονται όλες οι συσκευές, συνδέσεις και DNS εγγραφές από το αρχείο.

## ⌨️ ΣΥΝΤΟΜΕΥΣΕΙΣ ΠΛΗΚΤΡΟΛΟΓΙΟΥ

- **F1**: Άνοιγμα αναλυτικού βοηθητικού οδηγού (help mode) σε fullscreen κονσόλα.  
- **ESC**: Έξοδος από help mode / ακύρωση λειτουργιών.  
- **Home / End**: Πήγαινε στην αρχή / στο τέλος του κειμένου βοήθειας.  
- **PageUp / PageDown**: Κύλιση σελίδας στο help.  
- **↑ / ↓**: Λεπτή κύλιση στο help.  
- **DEL**: Διαγραφή συσκευής


***

## 🖥️ ΚΟΝΣΟΛΑ & DEBUG

- Κουμπί **Console**: Εμφάνιση/απόκρυψη της κονσόλας στο κάτω μέρος, με δυνατότητα αλλαγής ύψους με drag στη γραμμή τίτλου.  
- Η κονσόλα εμφανίζει logs για ping, route, DNS, debug και συστημικά μηνύματα.  
- Σε βασικές λειτουργίες (Ping, Route, DNS) η κονσόλα ανοίγει αυτόματα αν είναι κρυμμένη.  

- Κουμπί **Debug**: Ενεργοποιεί λεπτομερή logs (συνδέσεις, καταστάσεις συσκευών, σφάλματα) στην κονσόλα.

## 📊 ΣΤΑΤΙΣΤΙΚΑ ΔΙΚΤΥΟΥ (QUICK STATS)

Στο αριστερό κάτω μέρος εμφανίζονται:

## ⚠️ TROUBLESHOOTING

### 🔌 Χωρίς επικοινωνία / πρόβλημα συνδέσεων  
- Ελέγξτε IP, subnet και gateway των συσκευών.  
- Βεβαιωθείτε ότι βρίσκονται στο σωστό subnet.  
- Χρησιμοποιήστε το εργαλείο **Route** για να εντοπίσετε πού «σπάει» η διαδρομή.  

### 🌐 Προβλήματα με DNS  
- Ελέγξτε ότι η συσκευή έχει σωστό DNS server (π.χ. 192.168.1.53 ή 192.168.1.1).  
- Χρησιμοποιήστε **Manual DNS** για δοκιμή επίλυσης συγκεκριμένου domain.  
- Ελέγξτε αν το domain υπάρχει στα DNS records ή ως domainName κάποιας συσκευής.  

### ▶️ Η προσομοίωση δεν ξεκινά  
- Ελέγξτε ότι υπάρχουν τουλάχιστον δύο συσκευές συνδεδεμένες.  
- Βεβαιωθείτε ότι οι συσκευές είναι online και έχουν έγκυρες IP ρυθμίσεις.
